# v2.8.0-rc.7 — Ascension Runtime + Green Badges Checklist

## Purpose

This checklist packages two bounded acceptance surfaces:

1. README badge rail discipline with earned workflow status.
2. Bounded local/devnet Minimum Viable Ascension Runtime replay.

This checklist does **not** widen claims to audited-final deployment, mainnet safety, live global marketplace operation, or completed live Ascension.

## Acceptance criteria

- README badge rail is marker-managed and generated from `release/badges.json`.
- Dynamic workflow badges point to real workflow files:
  - `ci.yml`
  - `contracts-security.yml`
  - `release-provenance.yml`
- Badge rail validation passes and disallows forbidden claim inflation.
- Ascension runtime assert replay passes locally and emits required artifacts.
- Front-door docs keep explicit demonstrated/simulated/unproven boundaries.

## Commands

Run from repository root:

```bash
python3 scripts/generate_readme_badges.py --write
python3 scripts/check_readme_badges.py
python3 scripts/check_readme_badges.py --require-green-workflows
python3 demos/ascension-runtime/run_demo.py --assert
python3 scripts/check_ascension_runtime_artifacts.py
python scripts/check_math_markdown.py
python scripts/check_doctrine_consistency.py
python scripts/check_demo_links.py
python scripts/check_release_surface_posture.py
```

## Artifact existence checks

Use the deterministic artifact contract checker after `run_demo.py --assert`:

```bash
python3 scripts/check_ascension_runtime_artifacts.py
```

This verifies all required outputs for every Ascension layer, including scorecard and report surfaces.

## Workflow green requirement

The README workflow badges are considered earned only when the latest default-branch runs are passing for:

- CI
- Contracts security
- Release provenance

`release-provenance.yml` includes a non-release smoke path on push/pull_request and a release publication path on `workflow_dispatch`. The workflow badge reflects the latest run status and must remain green before publication.

## Schema checks

If Ascension runtime payload shape changes are introduced in the same cut:

- update relevant files under `schemas/v2.8/`
- validate updated payloads in `run_demo.py --assert`
- document migration implications in `CHANGELOG.md`

## Claim boundary

This release cut claims:

- badge governance hardening for truthful front-door workflow visibility;
- deterministic local/devnet Ascension runtime replay with machine-readable artifacts.

This release cut does **not** claim:

- audited-final deployment;
- mainnet-safe default operation;
- live DEX/token settlement;
- completed α‑AGI Ascension economy.

## Migration notes

- No contract interface migration is required.
- No backend database migration is required.
- Existing workflow-dispatch release provenance flow remains unchanged.

## Rollback notes

If release-provenance badge behavior regresses:

1. Revert `.github/workflows/release-provenance.yml` to last known green commit.
2. Re-run `python3 scripts/check_readme_badges.py --require-green-workflows`.
3. Restore forward fixes in a follow-up RC cut rather than weakening badge checks.
