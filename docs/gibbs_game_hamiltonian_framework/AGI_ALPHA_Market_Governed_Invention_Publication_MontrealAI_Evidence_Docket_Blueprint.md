# AGI ALPHA MontrealAI Evidence Docket Blueprint

Purpose: convert the MontrealAI GitHub implementation corpus into externally legible empirical evidence for AGI ALPHA as a scalable substrate for intelligence organizations.

## Claim boundary

The GitHub corpus supplies implementation, protocol, CI, local/devnet, demo, and proof-scaffold evidence. It does not, by itself, prove AGI, ASI, empirical SOTA, safe autonomy, or Kardashev-scale capability. Empirical claims require equal-budget benchmark comparisons, proof bundles, replay logs, validator reports, cost/risk ledgers, α-Work Unit calibration, and independent audit hooks.

## Benchmark ladder

- B0: repository text claims only.
- B1: demo run without evidence bundle.
- B2: CI logs only.
- B3: local proof bundle without baseline.
- B4: proof bundle with local replay and cost/safety ledgers.
- B5: full AGI ALPHA Evidence Docket with baselines, replay, validators, α-WU calibration, and independent reproduction instructions.

## First recommended task families

1. Software repair.
2. CI failure remediation.
3. Protocol/contract correctness.
4. OpenAPI/ABI export verification.
5. Documentation and runbook consistency.
6. Local/devnet replay validation.
7. AGI Jobs protocol-native tasks.
8. Node-runtime telemetry and sentinel checks.
9. Nova-Seeds proof-ladder demos.

## Promotion rule

B5 is promoted only if an independent reviewer can reproduce the run, replay the proof bundle, verify validator decisions, compare against baselines, inspect cost/risk ledgers, and confirm zero critical safety violations.
