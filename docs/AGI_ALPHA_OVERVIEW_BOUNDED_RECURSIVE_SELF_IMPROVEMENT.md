# AGI Alpha Overview

## Bounded Recursive Self-Improvement

**Doctrine, evidence, and claim boundary | April 2026**

**AGI Alpha is designed for bounded recursive self-improvement:** each completed mandate can produce a governed capability package that measurably improves adjacent future mandates under controlled evaluation.

AGI Alpha is being built to achieve bounded recursive self-improvement, and early internal evidence suggests the mechanism may be working within the protocol-correctness wedge.

## Core reading of the project

AGI Alpha is not another AI interface. It is a compounding invention substrate that aims to turn validated work into reusable future leverage.

The right near-term claim is a bounded one: governed package creation, adjacent transfer, and controlled measurement - not unconstrained self-redesign.

The right proof standard is also bounded: a real Mandate 1, a frozen package, a real adjacent Mandate 2, control vs treatment, and a scorecard against explicit thresholds.

## Current public posture

The public repository frames `alpha-nova-seeds` as a verifiable release candidate and production-grade starter architecture, not as an audited final deployment or a fully proven compounding substrate.

That posture is strategically correct. It preserves ambition while keeping the public claim boundary honest.

## What AGI Alpha is

AGI Alpha is a system for selecting frontier work, executing it, validating it, settling it, freezing the resulting capability into governed packages, and reusing those packages on adjacent work. Inside the operating doctrine, AGI Jobs is the execution and settlement kernel, while the archive is the memory surface that makes recursive improvement possible.

The fuller internal organism model can be read as:

`Insight -> Nova-Seeds -> MARK -> Sovereign -> AGI Jobs -> Archive`

### Bounded recursive self-improvement loop

`Mandate n -> Governed package -> Adjacent mandate n+1 -> Scorecard -> Archive`

Why the word **bounded** matters: package creation is governed, adjacent transfer is tested under fixed scope, improvement is measured against a published scorecard, and human or governance gates remain outside the package itself. This is why bounded recursive self-improvement is a stronger and more credible doctrine than open-ended self-improvement rhetoric.

| Term | Meaning |
|---|---|
| Recursive | Outputs from one mandate improve the next. |
| Self-improvement | Effective capability rises over time. |
| Bounded | Freeze, review, and governance gates constrain the loop. |
| Measurable | Control vs treatment makes the claim falsifiable. |

## Proof standard and first wedge

The project earns its compounding claim only if one completed mandate creates a frozen capability package that materially improves the next adjacent mandate under control conditions. Later, the same mechanism should show transfer beyond the original wedge with less human intervention.

The first wedge is protocol and smart-contract correctness. That wedge is strategically attractive because it offers formal or semi-formal verification surfaces, short review cycles, dense reusable primitives, and a clear archive trail for future reuse.

### Published adjacent-proof thresholds

| Metric | Pass threshold | Why it matters |
|---|---:|---|
| AOY uplift | >= 35% | Improvement must be economically meaningful. |
| Speed to first accepted output | >= 30% faster | Reuse should accelerate useful work. |
| Repair / rework | >= 40% lower | Better packages should reduce churn. |
| Evidence completeness | >= 20% higher | Treatment should improve auditability. |
| Safety regression | None | Improvement cannot come from weaker controls. |
| Package dependence | >= 30% | Wins should actually come from the frozen package. |

### What would count as line-crossing proof

1. A real Mandate 1 completed on live code or operations.
2. A frozen governed package with a clear manifest and hash.
3. A real adjacent Mandate 2 run in control and treatment.
4. Independent, preferably blinded adjudication against the same rubric.
5. A clear pass on the published thresholds, followed by replication.

## Current evidence and claim boundary

The strongest current evidence is a stricter internal rerun within the protocol-correctness wedge. The same frozen package, `GovernanceValidationPack-v1`, cleared the adjacent-reuse scorecard on two adjacent mandate pairs under a fixed internal rubric.

### What the current evidence supports

- A frozen package can be defined, versioned, and applied to adjacent review work.
- Within the first wedge, treatment materially outperformed control across two internal passes.
- The mechanism is strategically meaningful enough to justify the wedge and the proof plan.

### What is not yet proven

- Independent blinded replication.
- Parallel or randomized lane execution with separate operators.
- Cross-domain transfer beyond the protocol-correctness wedge.
- Low-handholding operation at meaningful scale.

### Best current reading

These results are stronger than a demo and weaker than final proof.

The mechanism appears real and strategically valuable inside the first wedge.

The broader public claim should remain **early internal evidence** until independent reruns exist.

## Best next milestone and recommended language

The single upgrade that would change the credibility of the evidence most is an independently adjudicated rerun of the same frozen package with separate operators and randomized or parallel lane order. A second independent pass on another adjacent pair would make the claim materially harder to dismiss.
