# Contracts risk matrix (v2.6.0-rc.1)

This matrix maps each core contract to the security properties currently covered by automated checks.

It is a hardening artifact for a **verifiable release candidate** and does **not** replace independent audit.

| Contract | Primary asset / authority surface | Most likely failure modes | Highest-severity invariants | Automated evidence (tests/properties) | Remaining out-of-scope risk |
|---|---|---|---|---|---|
| `AlphaNovaSeedV25` | Seed identity NFT mint + metadata authority via registry gate. | Unauthorized mint/URI mutation, registry hijack, token-ID drift. | Only registry can mint/mutate; registry cannot be zero; missing token URI must revert. | Hardhat: `contracts/test/alpha-seed.lifecycle.ts`; Foundry: `contracts/foundry-test/ContractsSecurity.t.sol`; Echidna interaction: `contracts/echidna/harnesses/EchidnaRegistryHarness.sol`. | NFT-level metadata trust is still dependent on off-chain URI immutability policy and operator controls. |
| `ChallengePolicyModuleV25` | Challenge policy activation + adjudication outcome logic. | Policy mismatch injection, inactive policy voting, double-finalization. | A challenge cannot cross policies; finalized adjudications cannot be mutated; only adjudicators can vote/finalize. | Hardhat: `contracts/test/challenge.policy.ts`; Foundry: `contracts/foundry-test/ContractsSecurity.t.sol`. | Weight/approval policy values still need human governance review for economic adequacy. |
| `CouncilGovernanceV25` | Seat assignment, election admin power, bonded challenge flow. | Unauthorized seat mutation, impossible challenge resolution, bond routing bugs. | Unknown/double challenge resolution impossible; upheld challenge deactivates seat; seat map has no active gaps. | Hardhat: `contracts/test/governance.lifecycle.ts`; Foundry unit+fuzz+invariants: `contracts/foundry-test/ContractsSecurity.t.sol`, `contracts/foundry-test/FuzzAndInvariant.t.sol`; Echidna: `contracts/echidna/harnesses/EchidnaGovernanceHarness.sol`. | Governance game-theory and cartel/censorship behavior remain audit/manual-review territory. |
| `NovaSeedRegistryV25` | Canonical seed state machine + review and sovereign registration anchors. | Duplicate seed insertion, invalid identity payload acceptance, unauthorized creator mutation, illegal terminal transition. | Draft requires valid identities; only creators/owner can mutate lifecycle; sovereign registration restricted to greenlit/blooming states. | Hardhat: `contracts/test/registry.lifecycle.ts`; Foundry: `contracts/foundry-test/ContractsSecurity.t.sol`, `contracts/foundry-test/FuzzAndInvariant.t.sol`; Echidna: `contracts/echidna/harnesses/EchidnaRegistryHarness.sol`. | Off-chain review quality and reviewer identity assurance are not guaranteed by on-chain checks alone. |
| `NovaSeedWorkflowAdapterV25` | Owner-mediated workflow bridge between registry state and assay jobs. | Unauthorized assay creation/finalization, bypass of greenlit/blooming gate. | Only owner can route workflow; assay creation blocked unless seed is greenlit or blooming. | Hardhat: `contracts/test/workflow.adapter.ts`; Foundry: `contracts/foundry-test/ContractsSecurity.t.sol`; Echidna: `contracts/echidna/harnesses/EchidnaWorkflowHarness.sol`. | External workflow engine correctness and job economic behavior remain integration/audit concerns. |

## Release-blocking interpretation

- **High severity finding** in Slither or any failing invariant/property in Foundry/Echidna is release-blocking until resolved.
- **Medium severity finding** requires explicit reviewer triage + mitigation plan before release candidate promotion.
- **Low severity finding** requires documented triage and backlog placement.
