# Support

## Operator support channels

- Open a GitHub Issue for bugs, docs gaps, or feature requests.
- Use issue templates when available.
- Include environment details (OS, Python, Node, chain RPC, DB version).

## What to include

- Expected behavior
- Actual behavior
- Reproduction steps
- Relevant logs/errors
- Version/commit hash

## Release candidate caveat

v2.6 is an RC hardening milestone. Some surfaces may still be evolving and are documented with explicit assumptions and non-goals.
