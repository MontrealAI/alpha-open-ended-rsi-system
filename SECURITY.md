# Security Policy

## Scope and posture

This repository tracks a **v2.6 release candidate**.
It is hardening-focused and **not represented as audited final deployment**.

## Reporting a vulnerability

Please report security issues privately via GitHub Security Advisories:

1. Go to the repository's **Security** tab.
2. Select **Report a vulnerability**.
3. Include impact, reproduction steps, and suggested mitigation.

If advisories are unavailable, open a private report to maintainers and include `SECURITY-REPORT` in the subject.

## Triage expectations

- Initial acknowledgement target: 3 business days.
- Triage classification target: 7 business days.
- Fix/release timing depends on severity and release windows.

## Safe harbor

Good-faith security research is welcome. Avoid data destruction, service disruption, and privacy violations.
